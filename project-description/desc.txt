Api, позволяющее пользователям оставлять ссылки на свои профили/каналы в социальных сетях

Стек: java 17, PostgreSQL, Spring Core, MVC, Boot, Data JPA, liquibase, docker

В моем приложении реализована связь "один ко многим" между сущностями "users" и "links".
Каждый пользователь может иметь несколько ссылок на свои ресурсы, но каждая ссылка принадлежит одному пользователю.
Для реализации этой связи использовались следующие поля:

users:
id(идентификатор пользователя)
username(пользовательское имя)
description(описание профиля пользователя)
created(дата создания пользователя)
changed(дата изменения информации пользователя(описание или имя))

links:
id(идентификатор ссылки)
service(сервис на котором располагается ресурс, например instagram, telegram)
link(сама ссылка на ресурс)
user_id(идентификатор пользователя, которому принадлежит ссылка)

SQL запросы по созданию таблиц приведены в миграциях в resources/db/changelog/migrations/
